Week 3 - 7 Dec, 2014

Name: Rabbit

One-Liner: Building project with Ant

Notable Tech: Apache Ant

Concept:

Last week two weeks (week 1, 2), I created a Java object-oriented project dealing with multithreads in a MVC framework. And this week, I was curious about automating builds. I looked into Apache Ant and Maven, and decided to try Ant, because Ant requires a more refined approach to the build, while Maven seems to do many aspects of it for you (which is great! Don't get me wrong. But for my purpose, I want to learn/ try the itty-bitty parts).

In addition, I've worked with Ant before, but on the side of, "oh. ok. this project is an ant project, and to run it do these x, y, z commands. Ok. I'll just trust this Ant magic to do what I want".

Product:

This week, I didn't touch/ refactor any of the Java code, but added/ played with the build.xml for Ant (also installed Ant with Homebrew).

Demo:

Tech Specs:

  • Java code from two weeks (week 1, 2)
  • Apache Ant for builds
  • Git branch (got more familiar)

Thoughts:

Ant builds turn out to be a lot easier than I thought. Configuring the build file was pretty straight forward. I find the build.xml file to be very neat, because the dependencies and the xml format all comes together (under this Ant magic) very well. I do have to say it was a little tedious to set it up, but once that's up and running, it was very easy and efficient for me to clean/compile/build/run the project!
