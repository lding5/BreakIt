Thoughts:

I have worked with MVC frameworks before, but not entirely on my own, and usually with Eclipse/ not from scratch. So I started this project by following a tutorial and looking up a lot of references that helped me import the right Java classes to get this framework to work. Overall, I enjoyed it a lot, because I had to rethink and reorganize the code to fit into an MVC structure. Also, I can see how having this framework is especially useful and needed in larger projects, because it modularizes and organizes the code in a structured manner.

Also, I've worked with Java Swingx before, and working it this week reminds me of how much of a hassle I still think it is... Haha. Well. I decided to use Java Swing to provide a quick and easy way to create a View for my MVC. I could've easily used JSPs, but since I haven't used Swingx for a while, I decided to refresh my memory with it and give it another try. But yeah, Java Swingx seems to requires so many lines of code to simply put an element in the GUI, and doesn't even look that good- at least the way it is right out of the box.
