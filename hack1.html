Winnie the Pooh

Java Multi-Threads

Java

Concept:

I remembered learning this in school, but I was never as comfortable with threads. So this week, I decided to take a shot at it and get better at it.

Product:

Story: There is a pot of honey, and there are honey eaters who eat at different speeds and nap at different lengths after they eat. This program models what happens (who eats when) and also who ate the last bit of honey from the honey jar.

Demo:

Sample input file (input.txt)

30
Winnie 5 1000
Tiger 3 2000
Piglet 2 1500

Tech Specs:

  • All code written in Java, emphasis on Threads, Runnable interfaces
  • File input:
    • int = how much honey is in the pot at the beginning
    • list of honey eaters specifications: name, eating pace, naping duration
  • Console output
  • Each honey eater is it's own separate Java thread.

Thoughts:

Threads are tricky and it took me a while to get it sorted out, hence the result is only a console output, rather than something fancier. I learn how to manage multiple threads, make sure when I want them to be sharing the same resource or not (took me a while to figure out how all the different threads can share the same honey pot), how to sleep them, and the various pitfalls I might encounter.

I am glad I worked on this topic though, because I am now more comfortable with the different concepts and terminologies associated with threads.

I also learned more about my own learning style. This past week, I found myself more at ease when I can go through an example to start me off, jump right in on my own code for a bit, go back and read more tutorial/ references, then back at the code. However, often times, I find myself caught reading too indepth in that 2nd reading session, because I just wanted to "know it all" before tackling on this subject so that I don't make a simple mistake that could be easily avoided. But for these projects, the purpose is realy to spit some product out quickly, thus, forcing me to learn as much as I can in a short amount of time. So I need to trying and cut down the amount of safety net reading, and try this other way of learning.
